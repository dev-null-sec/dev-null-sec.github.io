/轻量前端/07-页面练习/

页面练习 ​

HTML、CSS、布局、JavaScript、DOM、请求和 Vue 的基础语法先落到静态页面里。这里做一个“资料库”练习，页面使用本地 mock 数据，只读取 JSON 文件。

练习内容是一页“资料库”：左侧分类，右侧资料列表，顶部搜索，表单新增资料。数据刷新后会恢复到本地 JSON 的内容。

一、练习范围 ​

这个页面只练前端基础：

范围内容
页面结构头部、侧栏、内容区、列表、表单
样式布局Flex、卡片、表格、响应式
页面状态loading、error、empty
数据来源本地 JSON 文件
交互搜索、筛选、新增本地数据

数据只保存在浏览器变量里，刷新页面后，新增内容会丢失。

二、目录结构 ​

目录：

text
frontend-practice/
├── index.html
├── style.css
├── app.js
└── data/
    └── items.json

index.html 放结构，style.css 放样式，app.js 放交互，data/items.json 放本地数据。

六、页面逻辑 ​

文件：app.js

js
const statusText = document.querySelector("#status");
const list = document.querySelector("#list");
const empty = document.querySelector("#empty");
const searchInput = document.querySelector("#search-input");
const form = document.querySelector("#item-form");
const categoryButtons = document.querySelectorAll(".category-button");

let items = [];
let activeCategory = "all";
let keyword = "";

const categoryNames = {
  all: "全部",
  book: "图书",
  note: "笔记",
  link: "链接",
};

function getVisibleItems() {
  // 根据当前分类和搜索词计算列表，原始 items 不在这里改动。
  return items.filter((item) => {
    const matchCategory =
      activeCategory === "all" || item.category === activeCategory;
    const matchKeyword = item.title.includes(keyword);

    return matchCategory && matchKeyword;
  });
}

function renderItems() {
  const visibleItems = getVisibleItems();
  list.innerHTML = "";

  // hidden 为 true 时浏览器不显示空状态。
  empty.hidden = visibleItems.length > 0;

  for (const item of visibleItems) {
    const article = document.createElement("article");
    article.className = "item";

    const tag = document.createElement("span");
    tag.className = "tag";
    tag.textContent = categoryNames[item.category] || item.category;

    const title = document.createElement("h3");
    title.textContent = item.title;

    const description = document.createElement("p");
    description.textContent = item.description || "暂无备注";

    article.append(tag, title, description);
    list.appendChild(article);
  }
}

async function loadItems() {
  statusText.textContent = "加载中...";

  try {
    const response = await fetch("data/items.json");
    if (!response.ok) {
      throw new Error(`读取失败：${response.status}`);
    }

    items = await response.json();
    statusText.textContent = `共 ${items.length} 条`;
    renderItems();
  } catch (error) {
    // 把错误直接显示在页面上，方便确认是路径错、文件丢失还是 JSON 异常。
    statusText.textContent = error.message;
  }
}

for (const button of categoryButtons) {
  button.addEventListener("click", () => {
    activeCategory = button.dataset.category;

    for (const current of categoryButtons) {
      current.classList.toggle("active", current === button);
    }

    renderItems();
  });
}

searchInput.addEventListener("input", () => {
  keyword = searchInput.value.trim();
  renderItems();
});

form.addEventListener("submit", (event) => {
  event.preventDefault();

  const formData = new FormData(form);
  // Date.now() 只是本地临时 id，刷新页面后数据会从 JSON 重新读取。
  const item = {
    id: Date.now(),
    title: formData.get("title"),
    category: formData.get("category"),
    description: formData.get("description"),
  };

  items.unshift(item);
  form.reset();
  statusText.textContent = `共 ${items.length} 条`;
  renderItems();
});

loadItems();

items、activeCategory、keyword 是页面状态。渲染函数每次根据当前状态重新计算可见列表。数据量很小时，这种写法足够清楚。

七、运行方式 ​

这个页面需要读取本地 JSON，直接双击 HTML 文件时可能受浏览器限制。用静态文件服务打开目录更稳。

常见方式：

bash
npx serve .

或者使用编辑器自带的 Live Server。打开后访问提示里的本地地址。

如果不想安装工具，也可以把 items 直接写在 app.js 里，跳过 fetch：

js
let items = [
  { id: 1, title: "HTML 基础", category: "note", description: "页面结构" },
  { id: 2, title: "CSS 布局", category: "note", description: "页面样式" },
];

这种方式少了请求状态练习，但适合快速看页面。

八、常见问题 ​

现象常见原因排查入口
页面没有样式style.css 路径写错Network、Elements
数据读取失败没用静态文件服务、JSON 路径错Network
分类按钮没反应data-category 缺失或事件没绑定Elements、Console
搜索不生效keyword 没更新或渲染函数没调用input 事件
新增后不显示数据没 push/unshift，或被筛选条件过滤items、筛选条件
刷新后新增数据消失数据只存在浏览器内存里当前设计如此

这个练习处理浏览器里的临时数据和本地 JSON。刷新以后，新增内容会恢复到初始数据。

本地 JSON 文件适合练习页面逻辑，但数据不会持久保存，也没法让多个人共用同一份数据。页面里 fetch 的地址从 data/items.json 换成一个后端服务的 URL 后，数据的读写、保存和查询就由后端接口负责。后端接口的入口从 FastAPI 开始。
