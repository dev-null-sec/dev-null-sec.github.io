布局与响应式 ​

HTML 把内容放到页面里，CSS 还要决定这些内容怎么排列。标题在上面，导航在左侧还是顶部，内容区多宽，表格在小屏幕上会不会挤出屏幕，这些都属于布局问题。

布局从普通文档流、盒子尺寸、Flex、常见页面区域和响应式开始。Grid 也很常用，但轻量页面里 Flex 已经能覆盖大多数横向排列、纵向排列和简单双栏页面。

一、文档流 ​

浏览器默认会按 HTML 出现顺序摆放元素。块级元素通常从上到下排列，行内元素通常在一行里跟着文字流动。

html
<h1>资料整理</h1>
<p>第一段文字。</p>
<p>第二段文字。</p>
<a href="about.html">关于页面</a>
<a href="contact.html">联系信息</a>

h1 和 p 会各占一行，两个链接会排列在同一行。这个默认行为叫普通文档流。布局不是一开始就要写复杂 CSS，很多页面先利用文档流就能得到可读结构。

常见显示类型：

类型例子默认表现
blockdiv、p、section独占一行，可以设置宽高
inlinea、span、strong跟随文字排列，宽高不按块级方式生效
inline-block手动设置在一行内排列，也能设置宽高和内边距
flex手动设置子元素按弹性盒模型排列
css
.tag {
  display: inline-block;
  padding: 2px 8px;
  border: 1px solid #d1d5db;
  border-radius: 999px;
}

inline-block 常用于小标签、短状态、徽标这类内容。它不像普通行内元素那样难控制尺寸。

二、宽度高度 ​

页面布局里，宽度比高度更常控制。高度很多时候由内容自然撑开，强行固定高度反而容易出现文字溢出。

css
.page {
  max-width: 960px;
  margin: 0 auto;
  padding: 24px;
}

这段表示页面内容最多 960px 宽，左右自动留白，内部有 24px 间距。

常见宽度写法：

写法含义
width: 320px固定宽度
width: 100%占满父元素宽度
max-width: 960px最大宽度不超过 960px
min-width: 240px最小宽度不低于 240px

内容溢出时可以控制滚动：

css
.table-wrap {
  overflow-x: auto;
}

表格列很多时，外层加横向滚动比把文字压到看不清更稳。

三、Flex基础 ​

Flex 用来处理一组子元素的排列。给父元素设置 display: flex，它的直接子元素就进入 Flex 布局。

html
<div class="toolbar">
  <button type="button">新建</button>
  <button type="button">导入</button>
  <button type="button">导出</button>
</div>
css
.toolbar {
  display: flex;
  gap: 8px;
}

gap 控制子元素之间的间距，比给每个按钮单独写 margin-right 更清楚。

Flex 主轴方向：

css
.row {
  display: flex;
  flex-direction: row;
}

.column {
  display: flex;
  flex-direction: column;
}
属性常用值含义
displayflex开启 Flex
flex-directionrow / column横向或纵向排列
gap8px子元素间距
justify-contentspace-between、center主轴对齐
align-itemscenter、stretch交叉轴对齐
flex-wrapwrap子元素放不下时换行

横向工具栏：

css
.toolbar {
  display: flex;
  align-items: center;
  justify-content: space-between;
  gap: 12px;
}

左边标题、右边按钮：

html
<div class="toolbar">
  <h2>阅读安排</h2>
  <button type="button">新增</button>
</div>

justify-content: space-between 会把两端内容分开，常用于标题和操作按钮同一行。

四、弹性尺寸 ​

Flex 子元素可以用 flex 控制占用空间。

html
<div class="layout">
  <aside class="sidebar">目录</aside>
  <main class="content">正文</main>
</div>
css
.layout {
  display: flex;
  gap: 16px;
}

.sidebar {
  width: 220px;
  flex: none;
}

.content {
  flex: 1;
}

flex: none 表示侧栏保持固定宽度，flex: 1 表示内容区占用剩余空间。

几个常用写法：

写法含义
flex: none不伸缩，按宽度显示
flex: 1占用剩余空间
flex: 0 0 240px基础宽度 240px，不伸缩

卡片列表可以换行：

css
.cards {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}

.card {
  flex: 1 1 240px;
  min-width: 0;
}

flex: 1 1 240px 表示卡片基础宽度 240px，空间多时可以变宽，空间不足时可以收缩并换行。

五、常见页面结构 ​

简单页面经常由头部、侧栏、内容区组成：

html
<div class="shell">
  <header class="header">资料整理</header>
  <div class="body">
    <nav class="sidebar">目录</nav>
    <main class="content">正文内容</main>
  </div>
</div>
css
.shell {
  min-height: 100vh;
}

.header {
  padding: 16px 24px;
  border-bottom: 1px solid #e5e7eb;
}

.body {
  display: flex;
  min-height: calc(100vh - 57px);
}

.sidebar {
  width: 220px;
  flex: none;
  padding: 16px;
  border-right: 1px solid #e5e7eb;
}

.content {
  flex: 1;
  min-width: 0;
  padding: 24px;
}

min-width: 0 在 Flex 内容区里很常用。没有它时，长文本、表格、代码块可能把内容区撑出屏幕。

顶部导航结构：

css
.topbar {
  display: flex;
  align-items: center;
  justify-content: space-between;
  gap: 16px;
}

.nav {
  display: flex;
  gap: 12px;
}

头部、侧栏、内容区这些结构先用中性名字。具体视觉样式放到 CSS 里慢慢调整。

六、响应式 ​

响应式指同一个页面在不同屏幕宽度下调整布局。桌面屏可以左右分栏，小屏幕通常改成上下排列。

媒体查询写法：

css
@media (max-width: 720px) {
  .body {
    flex-direction: column;
  }

  .sidebar {
    width: auto;
    border-right: 0;
    border-bottom: 1px solid #e5e7eb;
  }
}

当视口宽度小于等于 720px 时，.body 从左右排列变成上下排列。

常见断点不是固定标准，可以按内容决定：

宽度常见处理
960px 以下减少左右留白
720px 以下双栏改成单栏
480px 以下按钮、表单、表格进一步压缩

响应式先看内容是否能读、能点、不会横向溢出。动画和细节可以晚一点处理。

七、内容溢出 ​

页面在小屏幕上最容易出现几类溢出：长单词、长链接、宽表格、固定宽度卡片。

长文本换行：

css
.content {
  overflow-wrap: anywhere;
}

宽表格横向滚动：

html
<div class="table-wrap">
  <table>
    <thead>
      <tr>
        <th>日期</th>
        <th>主题</th>
        <th>备注</th>
      </tr>
    </thead>
  </table>
</div>
css
.table-wrap {
  overflow-x: auto;
}

.table-wrap table {
  min-width: 560px;
}

固定宽度慎用：

css
.card {
  width: 320px;
  max-width: 100%;
}

max-width: 100% 可以避免卡片在窄屏幕上超过父容器。

九、常见问题 ​

现象常见原因排查入口
子元素没有横向排列父元素没有设置 display: flex父元素样式
内容区被表格撑宽Flex 子元素缺少 min-width: 0 或表格没有滚动容器.content、.table-wrap
小屏幕仍是双栏媒体查询条件没命中、CSS 顺序被覆盖DevTools 视口宽度、Styles
元素间距不一致混用了多个方向的 margingap、父子元素间距
固定宽度溢出写死 width 后没有 max-width元素尺寸

布局排查时，开发者工具里选中元素，看它的盒模型尺寸、父元素 display、是否命中媒体查询，通常能很快定位。
