/互联网与架构基础/05-前端和后端/

前端和后端 ​

打开一个网站时，屏幕上看到的页面属于前端。登录、保存、查询、统计这些动作，通常要交给后端处理。前端和后端之间用 HTTP 接口传数据，最常见的数据格式是 JSON。

把前端和后端分开看，排查问题会清楚很多。按钮点了没反应，可能是浏览器里的 JavaScript 报错；接口返回 500，通常要看后端日志；接口返回 200 但页面空白，可能是字段名和前端预期不一致。

一、浏览器里的前端 ​

前端运行在浏览器里。浏览器读取 HTML 生成页面结构，读取 CSS 生成样式，执行 JavaScript 处理交互和接口请求。

技术作用
HTML页面上有哪些内容
CSS页面长什么样
JavaScript点击、输入、弹窗、接口请求

一个按钮能正常工作，要同时满足几件事：按钮在 HTML 里存在，CSS 没把它遮住，JavaScript 绑定了点击事件，点击后发出的接口请求也正常返回。

前端源码通常会被打包成浏览器能加载的静态文件。发布时常见产物是 index.html、assets/*.js、assets/*.css、图片和字体。这些文件可以放在 Nginx、对象存储或 CDN 上。

二、服务器里的后端 ​

后端服务运行在服务器上，负责处理请求、访问数据库、调用其他服务、返回结果。Python 的 FastAPI、Go 的 HTTP 服务、Java 的 Spring Boot、Node.js 的 Express，都可以作为后端服务。

后端里常见对象：

对象含义
路由把请求路径交给对应函数处理
处理函数读取参数、做业务处理、返回结果
模型描述业务对象或数据库表
数据库连接读写 MySQL、PostgreSQL、Redis 等
中间件处理认证、日志、跨域、异常

后端程序一般监听一个端口，例如 8080。公网用户通常不会直接访问这个端口，而是经过 Nginx、负载均衡或网关转发进来。

三、JSON 接口 ​

前后端分离后，后端常返回 JSON。JSON 是一种文本格式，能表达对象、列表、字符串、数字和布尔值。

一个任务列表接口可能返回：

json
{
  "items": [
    {"id": 1, "name": "巡检 web-01", "status": "success"},
    {"id": 2, "name": "备份 mysql-01", "status": "running"}
  ],
  "total": 2
}

前端拿到这段数据后，把 items 渲染成表格或卡片。后端要保证字段名、类型和含义稳定，前端才能按固定方式展示。

字段不一致时，页面会出现很具体的问题。比如后端返回 task_status，前端代码读 status，Network 面板里接口状态码是 200，但页面状态列可能是空的。这种问题不在 Nginx，也不在数据库连接，而在接口字段没有对齐。

四、分开部署 ​

前后端分离以后，静态页面和后端接口可以独立部署。

浏览器第一次请求拿到 index.html 和 JS 文件；JS 运行后再请求 /api/...。同一个页面可能同时访问静态资源域名和接口域名。

常见位置：

内容常见位置
前端静态文件Nginx、对象存储、CDN
后端 API虚拟机、容器、Kubernetes
数据库云数据库、自建数据库、集群
入口层Nginx、API 网关、Ingress

五、跨域 ​

浏览器有同源策略。协议、域名、端口任意一个不同，就属于不同源。例如 https://console.example.com 调用 https://api.example.com，域名不同，浏览器会按跨域请求处理。

后端需要返回正确的 CORS Header，浏览器才允许前端代码读取响应。跨域失败时，后端可能已经正常返回数据，但浏览器拦住了 JavaScript 读取响应的动作。

跨域报错常见在浏览器控制台里。一个典型现象是 Network 面板里能看到预检请求 OPTIONS，但响应头里没有允许的 Access-Control-Allow-Origin，页面接口调用失败。

六、联调时看哪里 ​

前后端联调最容易混的是“接口通了”和“页面可用”。

现象更接近的位置
接口返回 401登录状态、Token、Cookie
接口返回 404API 路径、网关转发、后端路由
接口返回 500后端应用、数据库、下游服务
接口 200 但页面空字段名、数据结构、前端渲染逻辑
控制台 CORS 报错跨域配置
JS 文件 404前端构建路径、静态资源发布路径

前端和后端通过 HTTP 接口接在一起。页面问题要看浏览器里的请求和控制台，后端问题要看应用日志、接口状态码、数据库和下游调用。
