一次访问网站 ​

这部分是概念扫盲，不深入复杂原理、部署细节和代码。尺度放得简单一点：把网站背后常见的东西认出来，知道它们大概站在哪一层。再遇到 Linux、Nginx、数据库、前端、后端、Kubernetes、监控和发布工具时，就不至于只看到一堆分散的名字。

一个网站能被打开，背后要解决很多朴素问题：外面的人怎么找到这台服务器，浏览器怎么和服务器说话，页面从哪里来，数据放在哪里，访问的人多了怎么办，机器坏了怎么办，出问题以后去哪里找证据。公网 IP、域名、HTTP、数据库、负载均衡、高可用、容灾、日志和监控，都是围绕这些问题出现的。

一次网站访问刚好能把这些东西串起来。浏览器打不开、接口超时、页面白屏、登录后一直转圈，表面看都是“网站有问题”，背后可能是域名解析错了，也可能是端口没放行、证书过期、Nginx 转发失败、后端报错、数据库变慢。

浏览器地址栏里输入 https://www.example.com/docs/index.html 后，浏览器会先把这一串地址拆开：使用 HTTPS，访问的名字是 www.example.com，路径是 /docs/index.html，默认端口是 443。后面的 DNS、IP、端口、HTTP、前端资源、后端接口，都从这串地址开始。

一、地址栏里的地址 ​

地址栏里的完整地址叫 URL。它有固定格式，里面写着几件事：用什么协议访问，访问哪个名字，连哪个端口，要取哪个路径下的内容。

以这个地址为例：

https://www.example.com:443/docs/index.html?from=notes#title

部分示例含义
协议https用 HTTPS 访问，通信会加密
域名www.example.com这个名字需要先换成 IP 地址
端口443HTTPS 默认端口
路径/docs/index.html要访问的页面或资源位置
查询参数from=notes传给服务端的附加信息
片段#title浏览器跳到页面里的某个位置

端口经常不会写在地址里。https://www.example.com/docs 看起来没有端口，浏览器仍然会按 HTTPS 默认值连接 443。如果是 http://，默认端口就是 80。

域名和路径要分开看。DNS 只负责把 www.example.com 这个名字换成 IP，不处理 /docs/index.html。路径是在连接到服务器以后，由 Nginx、Web 服务或后端程序继续处理的。

二、名字变成 IP ​

计算机之间实际通信靠 IP 地址。域名是给人看的名字，DNS 负责把名字换成 IP。

访问 www.example.com 时，浏览器和操作系统会去查这个名字对应哪个地址。最后可能拿到一条或多条记录：

记录示例含义
Awww.example.com -> 203.0.113.10域名指向 IPv4 地址
AAAAwww.example.com -> 2001:db8::10域名指向 IPv6 地址
CNAMEwww.example.com -> cdn.example.net域名先指向另一个域名

很多网站前面有 CDN。业务域名不一定直接指向源站服务器，而是先指向 CDN 厂商分配的域名，再由 CDN 返回一个离访问者更近的边缘节点 IP。浏览器看到的仍然是一个 IP 和端口，只是这个 IP 可能属于 CDN。

域名解析失败时，浏览器常见错误是 DNS_PROBE_FINISHED_NXDOMAIN。这类错误还没走到服务器和 HTTP 层，通常要看域名有没有写错、解析记录有没有配、DNS 缓存是不是还没更新。

三、连接 IP 和端口 ​

拿到 IP 后，浏览器会连接 IP:端口。HTTPS 常见是 203.0.113.10:443，HTTP 常见是 203.0.113.10:80。

端口用来区分同一台机器上的不同服务。同一个公网 IP 上可以同时有 SSH、HTTP、HTTPS、数据库代理等服务，访问哪个服务要看端口。

端口常见服务
22SSH
80HTTP
443HTTPS
3306MySQL
6379Redis

连接建立前会发生 TCP 三次握手。HTTPS 还会继续做 TLS 握手，用证书证明站点身份，并协商加密通信需要的密钥。证书过期、域名和证书不匹配、客户端不信任证书，都会在这一步附近暴露出来。

端口层面的错误和 DNS 错误不一样。ERR_CONNECTION_REFUSED 通常表示已经到达目标 IP，但目标端口没有服务在监听，或者服务主动拒绝。ERR_CONNECTION_TIMED_OUT 更像是中间没有回应，可能是公网 IP 不通、安全组没放行、防火墙丢包、路由异常，或者目标机器没有接到这个连接。

四、浏览器发请求 ​

连接建立以后，浏览器才开始发送 HTTP 请求。HTTP 可以理解成浏览器和服务端之间说话的格式：浏览器说“我要这个路径”，服务端回“这是结果”。

访问 https://www.example.com/docs/index.html 时，请求大概会包含这些信息：

请求内容作用
方法GET 读取资源，POST 提交数据
路径告诉服务端要访问哪个页面或接口
Host告诉入口层这次请求访问的是哪个域名
Header携带浏览器类型、Cookie、认证信息、内容类型等
Body表单、JSON、文件上传这类请求会带请求体

同一个入口 IP 后面可能挂着多个站点。Host 很重要，因为 Nginx 或网关会根据它判断请求属于哪个站点。比如 www.example.com 走官网，api.example.com 走接口服务，它们可能都经过同一个入口层。

能拿到 404、500、502 这类 HTTP 状态码，说明请求至少已经进入了 Web 入口或后端链路。DNS 解析失败和端口连接失败，通常还拿不到这些状态码。

五、请求进服务器 ​

公网请求到达机房或云厂商网络后，通常不会直接落到业务程序。中间可能经过 CDN、WAF、负载均衡、Nginx、Ingress、API 网关，再转给后端应用。

小站点可能只有一台云服务器和一个 Nginx；企业系统可能有 CDN、WAF、负载均衡、Kubernetes Ingress 和多个机房。链路越长，入口能力越多，排查时也要更清楚请求停在哪一层。

几个常见状态码能帮忙定位方向：

现象常见含义
404 Not Found请求到了服务端，但路径没有匹配到页面或接口
401 Unauthorized没登录，或者认证信息无效
403 Forbidden识别了身份，但没有权限
500 Internal Server Error后端程序处理请求时报错
502 Bad Gateway入口层连接后端失败
503 Service Unavailable后端暂时不可用，可能实例都不健康或服务过载

500 和 502 容易混。500 多半是后端已经收到请求，但处理时异常。502 更常见于入口层没有连上后端，比如 upstream 地址错了、Pod 没起来、端口配置不一致、后端连接被拒绝。

六、页面不是一次拿完 ​

服务端处理完请求后，会返回状态码、响应头和响应体。浏览器拿到 HTML 后，开始解析页面结构。

HTML 里通常还会引用 CSS、JavaScript、图片和字体。浏览器解析到这些资源时，会继续发新的请求。一个网页看起来只打开了一次地址，实际可能产生几十个请求。

资源浏览器拿来做什么
HTML页面结构
CSS字体、颜色、间距、布局
JavaScript页面交互、调用接口、更新内容
图片/字体页面素材
API JSON动态数据，例如用户信息、列表、统计值

前后端分离的页面里，第一次请求可能只拿到一个空壳 HTML 和几份静态资源。JavaScript 运行后再请求 /api/tasks、/api/users/me 这类接口，后端返回 JSON，浏览器再把数据填进页面。

页面加载慢也要拆开看。HTML 慢，可能是入口层、后端渲染或网络延迟；某个 API 慢，重点会转向后端函数、数据库查询、缓存和下游服务；图片和 JS 慢，则可能是静态资源、CDN、缓存策略或文件体积问题。
