数据库基础 ​

FastAPI 本身不负责保存数据。HTTP 请求进来以后，后端通常要把数据读写到数据库里。

直接写 SQL（比如用 pymysql 执行 SELECT / INSERT）能完成读写，但接口数量多起来后，SQL 字符串散落在函数里，字段名、表名、SQL 语法错误都要运行时才能发现。SQLAlchemy 是 Python 常用 ORM，它把 Python 类映射到数据库表，把对象操作转换成 SQL。字段名变成类属性后，写错时编辑器和类型检查能提前提示；查询条件可以用 Python 链式调用拼接，不用手工拼 WHERE 子句。

示例数据库使用 SQLite，内容覆盖连接、表模型、Session、CRUD 和 FastAPI 依赖。SQLite 不需要单独安装数据库服务，能把注意力放在 ORM 的工作方式上。MySQL 主要差异在连接地址、驱动和连接池参数。

一、依赖 ​

安装 SQLAlchemy：

bash
uv add sqlalchemy

项目目录：

text

└── app/
    ├── database.py
    ├── deps.py

    ├── schemas.py
    └── routers/

几个文件的分工：

文件内容
database.py数据库连接、Session 工厂、建表函数
models.pySQLAlchemy ORM 表模型
schemas.pyPydantic 请求和响应模型

routers/articles.py文章接口

二、连接配置 ​

新增文件：app/database.py

python
from sqlalchemy import create_engine
from sqlalchemy.orm import DeclarativeBase, sessionmaker


DATABASE_URL = "sqlite:///./app.db"


class Base(DeclarativeBase):
    pass


engine = create_engine(
    DATABASE_URL,
    connect_args={"check_same_thread": False},
)

SessionLocal = sessionmaker(
    bind=engine,
    autoflush=False,
    autocommit=False,
)

sqlite:///./app.db 表示数据库文件放在当前目录的 app.db。check_same_thread=False 是 SQLite 在 Web 服务里常见配置，否则一个连接跨线程使用时会报错。MySQL、PostgreSQL 不需要这个参数。

SessionLocal 是 Session 工厂。Session 不是数据库本身，它更像一次请求里的数据库操作上下文：查询、添加、提交、回滚都从这里发出去。每次请求创建一个 Session，用完关闭。

三、表模型 ​

新增文件：app/models.py

python
from datetime import datetime

from sqlalchemy import Boolean, DateTime, Integer, String, Text
from sqlalchemy.orm import Mapped, mapped_column

from app.database import Base


class Article(Base):
    __tablename__ = "articles"

    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    title: Mapped[str] = mapped_column(String(100), nullable=False)
    content: Mapped[str] = mapped_column(Text, nullable=False)
    published: Mapped[bool] = mapped_column(Boolean, default=False, nullable=False)
    created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.utcnow, nullable=False)

Article 映射到 articles 表。Mapped[...] 和 mapped_column() 是 SQLAlchemy 2.0 常用写法，类型标注和字段定义放在一起。

几个字段含义：

字段数据库列
id主键
title最长 100 字符，不能为空
content长文本，不能为空
published是否发布，默认 false
created_at创建时间

nullable=False 表示数据库层不允许为空。Pydantic 请求模型的校验挡在入口，数据库约束挡在持久化层，两层作用不同。

四、建表 ​

开发阶段可以用 Base.metadata.create_all() 创建表。

修改文件：app/database.py

python
def create_db_and_tables():
    # 开发阶段临时建表；正式维护表结构时交给 Alembic 迁移。
    Base.metadata.create_all(bind=engine)

修改文件：app/main.py

python
from contextlib import asynccontextmanager

from fastapi import FastAPI

from app import models
from app.database import create_db_and_tables
from app.routers import articles


@asynccontextmanager
async def lifespan(app: FastAPI):
    create_db_and_tables()
    yield


app = FastAPI(title="FastAPI Basic", lifespan=lifespan)
app.include_router(articles.router)

from app import models 这行看起来没有直接使用，但它会让 Article 模型被导入，Base.metadata 才能知道有哪些表。模型没有导入时，create_all() 可能什么表都不建。

五、Session依赖 ​

新增文件：app/deps.py

python
from collections.abc import Generator

from sqlalchemy.orm import Session

from app.database import SessionLocal


def get_db() -> Generator[Session, None, None]:
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

这个依赖函数为每个请求创建一个 Session，请求处理结束后关闭。yield 前是进入请求时执行，finally 是请求结束后执行。数据库连接不关闭时，服务跑久了会耗尽连接。

六、请求和响应模型 ​

新增文件：app/schemas.py

python
from datetime import datetime

from pydantic import BaseModel, Field


class ArticleCreate(BaseModel):
    title: str = Field(min_length=1, max_length=100)
    content: str = Field(min_length=1)
    published: bool = False


class ArticleRead(BaseModel):
    id: int
    title: str
    content: str
    published: bool
    created_at: datetime

SQLAlchemy 模型对应数据库表，Pydantic 模型对应 API 输入输出。两者分开后，数据库字段变化和接口字段变化不会完全绑死。比如数据库里可以有内部字段，响应模型里不返回。

七、创建数据 ​

新增文件：app/routers/articles.py

python
from typing import Annotated

from fastapi import APIRouter, Depends, status
from sqlalchemy.orm import Session

from app import models, schemas
from app.deps import get_db


router = APIRouter(prefix="/articles", tags=["articles"])


@router.post("", response_model=schemas.ArticleRead, status_code=status.HTTP_201_CREATED)
def create_article(
    payload: schemas.ArticleCreate,
    db: Annotated[Session, Depends(get_db)],
):
    article = models.Article(
        title=payload.title,
        content=payload.content,
        published=payload.published,
    )

    db.add(article)
    db.commit()
    db.refresh(article)

    return article

db.add()、db.commit()、db.refresh() 三步各有不同作用：

  • 之前：article 是一个内存里的 Python 对象，数据库里还没有这条记录，id 和 created_at 是空的
  • db.add()：把对象注册到当前 Session，告诉 SQLAlchemy"提交事务时把这个对象 INSERT 到数据库"
  • db.commit()：提交事务，把 INSERT 发给数据库执行。article.id 和 created_at 在数据库里生成，但内存里的 article 对象现在还不知道这些值
  • db.refresh()：从数据库重新读取刚 INSERT 的记录，回填到 article 对象里
  • 之后：article.id 有值（比如 1），article.created_at 有值，可以直接返回给客户端
  • 验证：提交后打印 article.id，或直接看返回的 JSON 里有没有 id 字段

八、读取列表 ​

继续修改文件：app/routers/articles.py

python
from sqlalchemy import select


@router.get("", response_model=list[schemas.ArticleRead])
def list_articles(
    db: Annotated[Session, Depends(get_db)],
    offset: int = 0,
    limit: int = 20,
):
    statement = (
        select(models.Article)
        .order_by(models.Article.id.desc())
        .offset(offset)
        .limit(limit)
    )
    return db.scalars(statement).all()

select(models.Article) 生成查询语句。db.scalars(statement).all() 返回 ORM 对象列表。offset 和 limit 是分页雏形，列表接口可以继续加范围校验。

九、读取单条 ​

python
from fastapi import HTTPException


@router.get("/{article_id}", response_model=schemas.ArticleRead)
def get_article(
    article_id: int,
    db: Annotated[Session, Depends(get_db)],
):
    article = db.get(models.Article, article_id)
    if article is None:
        raise HTTPException(status_code=404, detail="article not found")

    return article

db.get(Model, primary_key) 按主键查询。查不到返回 None，接口层转换成 404。

十、更新和删除 ​

更新：

python
@router.put("/{article_id}", response_model=schemas.ArticleRead)
def update_article(
    article_id: int,
    payload: schemas.ArticleCreate,
    db: Annotated[Session, Depends(get_db)],
):
    article = db.get(models.Article, article_id)
    if article is None:
        raise HTTPException(status_code=404, detail="article not found")

    article.title = payload.title
    article.content = payload.content
    article.published = payload.published

    db.commit()
    db.refresh(article)
    return article

删除：

python
@router.delete("/{article_id}", status_code=204)
def delete_article(
    article_id: int,
    db: Annotated[Session, Depends(get_db)],
):
    article = db.get(models.Article, article_id)
    if article is None:
        raise HTTPException(status_code=404, detail="article not found")

    db.delete(article)
    db.commit()
    return None

db.delete() 把对象标记为待删除，db.commit() 执行 DELETE 语句。之前数据库里有一条 id=1 的记录，之后该记录不存在。验证：再次 GET /articles/1 应返回 404。

204 表示删除成功但没有响应体。返回 None 时，客户端只关心状态码。

十一、MySQL连接 ​

SQLite 适合本地学习和小工具。接 MySQL 时需要安装驱动：

bash
uv add pymysql

连接地址示例：

python
DATABASE_URL = "mysql+pymysql://app_user:app_password@127.0.0.1:3306/app_db?charset=utf8mb4"

部署配置通常不写死在代码里，而是从环境变量读取：

python
import os


DATABASE_URL = os.environ["DATABASE_URL"]

环境变量缺失时，服务启动阶段直接失败，比运行到第一个请求才报数据库连接错误更清楚。

十二、常见问题 ​

现象常见原因排查入口
表没有创建模型没有导入，或 create_all() 没执行启动日志、数据库表
返回 500Session 没提交、字段为空、数据库约束失败终端日志
自增 id 为空commit() 后没有 refresh()创建接口
SQLite 线程错误缺少 check_same_thread=Falsedatabase.py
MySQL 连不上地址、账号、密码、库名、驱动错误DATABASE_URL
连接耗尽Session 没关闭get_db()

数据库问题先区分两层：应用有没有正确创建 Session，数据库本身能不能连接和执行 SQL。FastAPI 的函数只是入口，实际的数据读写发生在 SQLAlchemy Session 里。
