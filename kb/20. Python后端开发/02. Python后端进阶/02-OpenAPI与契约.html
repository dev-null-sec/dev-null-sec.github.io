OpenAPI与契约 ​

路径、方法、状态码和错误格式固定下来以后，还需要一份能被人和工具共同读取的接口说明。只靠口头约定时，字段改名、必填项变化、认证方式变化很容易在联调时才暴露。

OpenAPI 是接口契约的一种机器可读格式。FastAPI 会根据路由、参数、Pydantic 模型和响应模型生成 /openapi.json，再用 /docs 展示出来。字段名、类型、必填项、状态码、认证方式和示例都写进文档后，前端、脚本、测试工具和后端可以围绕同一份契约对齐。

一、契约来源 ​

FastAPI 的文档信息主要来自几个地方：

来源文档里体现
路由路径/articles/{article_id}
HTTP 方法GET、POST
参数类型path、query、header、body
Pydantic 模型请求体和响应体 schema
response_model成功响应结构
responses额外状态码说明
tags文档分组

示例：

python
router = APIRouter(prefix="/articles", tags=["articles"])


@router.get(
    "/{article_id}",
    response_model=ArticleRead,
    responses={404: {"description": "Article not found"}},
)
def get_article(article_id: int):
    ...

这段会让文档里出现 GET /articles/{article_id}、路径参数 article_id、成功响应模型和 404 说明。

二、模型说明 ​

Pydantic 字段可以加说明和示例。

python
from pydantic import BaseModel, Field


class ArticleCreate(BaseModel):
    title: str = Field(
        min_length=1,
        max_length=100,
        description="文章标题",
        examples=["FastAPI 基础"],
    )
    content: str = Field(
        min_length=1,
        description="文章正文",
    )

字段说明会进入 OpenAPI。字段名不够自解释时，description 能减少联调时的误解。

三、响应模型 ​

请求模型和响应模型分开。

python
class ArticleRead(BaseModel):
    id: int
    title: str
    content: str
    published: bool


@router.post("", response_model=ArticleRead, status_code=201)
def create_article(payload: ArticleCreate):
    ...

response_model 有两个作用：文档里显示响应结构，运行时过滤返回字段。内部字段不应该出现在响应里时，响应模型就是最后一层保护。

四、额外响应 ​

接口可能返回不止一种状态码。

python
@router.get(
    "/{article_id}",
    response_model=ArticleRead,
    responses={
        404: {
            "description": "Article not found",
            "content": {
                "application/json": {
                    "example": {
                        "error": {
                            "code": "article_not_found",
                            "message": "article not found",
                        }
                    }
                }
            },
        }
    },
)
def get_article(article_id: int):
    ...

这类补充适合记录业务错误。422 这类校验错误 FastAPI 会自动生成，业务错误需要接口自己说明。

五、认证文档 ​

使用 OAuth2PasswordBearer 后，OpenAPI 会知道接口需要 Bearer Token。

python
from typing import Annotated

from fastapi import Depends
from fastapi.security import OAuth2PasswordBearer


oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/token")


@router.get("/me")
def read_me(token: Annotated[str, Depends(oauth2_scheme)]):
    return {"token": token}

/docs 右上角会出现授权入口。调用保护接口前，可以在文档里填入 token 进行调试。

六、导出契约 ​

OpenAPI JSON 地址：

text
http://127.0.0.1:8000/openapi.json

导出：

bash
curl http://127.0.0.1:8000/openapi.json -o openapi.json

导出的 openapi.json 可以用于：

用途说明
前端生成类型根据 schema 生成 TypeScript 类型
接口测试导入 Postman、Insomnia 等工具
契约评审对比字段、状态码、路径变化
网关配置某些 API 网关能读取 OpenAPI

契约文件保存后，也可以在发布前对比差异。字段删除、类型变化、状态码变化都属于需要关注的接口变更。

七、命名约定 ​

字段名建议保持一种风格。JSON 里常见小驼峰或下划线：

json
{
  "created_at": "2026-01-01T00:00:00Z"
}

或：

json
{
  "createdAt": "2026-01-01T00:00:00Z"
}

同一个项目里混用会增加前端和脚本处理成本。Python 后端常用下划线字段，前端常用小驼峰；具体选哪种要在契约里固定。

八、变更记录 ​

接口变化大致分两类：

类型例子影响
兼容变更新增可选字段、新增接口调用方通常不受影响
不兼容变更删除字段、改字段类型、改状态码含义调用方需要同步修改

OpenAPI 契约能帮助发现变化，但不能替代变更说明。特别是字段含义变化，schema 可能看不出来，文档说明仍然要写清楚。

九、常见问题 ​

现象常见原因排查入口
文档里响应是空对象接口缺少 response_model路由装饰器
参数说明不清楚没有 Field/Query 描述Pydantic 模型、参数
错误状态码没显示没有配置 responses路由装饰器
登录按钮不出现没接入 security dependencyOAuth2 配置
契约和实际返回不同返回对象绕过了响应模型，或模型没更新response_model、实际响应

OpenAPI 是接口契约的自动生成结果。路由和模型写得越明确，契约越可靠；代码里大量返回裸 dict，契约就会退化成粗略说明。
